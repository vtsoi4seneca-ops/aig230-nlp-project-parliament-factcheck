# This is an NLP project for AIG230
Building an NLP system to fact-check Canadian parliamentary debates using historical Hansard records (1901-present).\
The project will use PostgreSQL database with the debate records and will fine-tune an LLM to verify factual claims made by MPs.

## File Descriptions

### Configuration Files

| File | Purpose |
|------|---------|
| `pyproject.toml` | Python project metadata and dependencies. Uses `uv` package manager with PyTorch CUDA 13.0 for DGX Spark ARM64 |
| `uv.lock` | Exact dependency lockfile ensuring reproducible builds across DGX Spark nodes |
| `docker-compose.yml` | Orchestrates PostgreSQL database and fact-checking API service |
| `Dockerfile` | Container build for training environment |
| `Dockerfile.api` | Container build for inference API service |
| `activate.sh` | Shell script to activate virtual environment and set CUDA variables |

### Data Directory (`data/`)

| File | Generated By | Purpose |
|------|--------------|---------|
| `extracted_claims.json` | `extract_claims.py` | Raw factual claims extracted from Hansard debates using pattern matching |
| `labeled_claims.json` | `label_claims.py` or `labeling_tool.py` | Human-verified claims with TRUE/FALSE/MISLEADING labels |
| `training_pairs.json` | `create_synthetic_data.py` or `label_claims.py` | Formatted prompt-completion pairs for LLM fine-tuning |

### Source Scripts (`src/`)

#### Data Pipeline

| Script | Usage | Description |
|--------|-------|-------------|
| `data_loader.py` | Imported by other scripts | Database interface classes: `HansardDB` (PostgreSQL connection) and `DebateProcessor` (NLP processing) |
| `extract_claims.py` | `uv run python src/extract_claims.py --limit 100` | Extracts factual claims from parliamentary debates using regex patterns and spaCy. Outputs to `data/extracted_claims.json` |
| `label_claims.py` | `uv run python src/label_claims.py` | Interactive CLI tool for manual fact-checking. Labels claims as TRUE/FALSE/MISLEADING/UNVERIFIED |
| `create_synthetic_data.py` | `uv run python src/create_synthetic_data.py` | Generates synthetic training data for initial model testing (not for production) |
| `labeling_tool.py` | `uv run python src/labeling_tool.py` | Streamlit web interface for collaborative claim labeling |

#### Model Training & Inference

| Script | Usage | Description |
|--------|-------|-------------|
| `train.py` | `uv run python src/train.py` | Fine-tunes LLaMA model using LoRA/QLoRA on labeled claims. Configured for DGX Spark 128GB unified memory |
| `api.py` | `uv run python src/api.py` | FastAPI service providing `/fact-check` endpoint for real-time claim verification |
| `evidence_retriever.py` | Imported by `api.py` | Retrieves supporting evidence from parliamentary records and government databases |

### Execution Pipeline

```bash
# Step 1: Extract claims from database
uv run python src/extract_claims.py --limit 1000 --start-date 2024-01-01

# Step 2: Label claims manually (CLI)
uv run python src/label_claims.py

# Step 3: Train model
uv run python src/train.py

# Step 4: Start API
uv run python src/api.py
# Or with Docker
docker-compose up -d
